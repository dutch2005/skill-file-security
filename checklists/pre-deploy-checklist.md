# 🔐 Pre-Deploy Security Checklist

> Run before every production deployment. Check all applicable items.
> Generated by security-skill v1.0.0

## 🔴 Critical — Block Deploy if Failed

- [ ] No `.env` files committed to git (`git ls-files | grep .env`)
- [ ] No hardcoded secrets in source code
- [ ] All API keys are environment variables, not hardcoded
- [ ] `NEXT_PUBLIC_*` prefix not used on secret values
- [ ] Database rules are NOT set to `allow all`
- [ ] Firebase: Firestore/Storage rules are not public
- [ ] Supabase: RLS enabled on all tables
- [ ] No `NODE_TLS_REJECT_UNAUTHORIZED = '0'` in code
- [ ] No `process.env` printed to console
- [ ] Authentication required on all protected routes
- [ ] JWT secret is >= 32 bytes random string

## 🟠 High — Fix Before Deploy

- [ ] CORS not using wildcard `*` in production
- [ ] Rate limiting configured on all API routes
- [ ] Security headers configured (HSTS, CSP, X-Frame-Options...)
- [ ] HTTPS enforced (no HTTP in production)
- [ ] Session cookies have: httpOnly, secure, sameSite=strict
- [ ] `npm audit` shows no critical/high vulnerabilities
- [ ] Source maps disabled in production build
- [ ] Admin/debug routes not accessible in production
- [ ] Error messages don't expose internal details to users
- [ ] File upload: MIME validation + size limits in place

## 🟡 Medium — Fix Soon After Deploy

- [ ] `.env.example` exists and is up to date
- [ ] `dependabot.yml` configured for automated updates
- [ ] `.gitignore` covers all sensitive file patterns
- [ ] DKIM/SPF/DMARC configured for sending domain
- [ ] security.txt file exists at `/.well-known/security.txt`
- [ ] `memory-security.md` created and up to date
- [ ] No packages with known vulnerabilities (audit done)
- [ ] Cookie banner present if using analytics/marketing cookies (GDPR)
- [ ] Privacy policy page exists

## 🔵 Low / Info — Track and Improve

- [ ] Secret rotation schedule defined (90 days recommended)
- [ ] GitHub Actions: all action versions pinned to SHA
- [ ] Docker images not running as root
- [ ] Pre-commit hooks set up for secret scanning
- [ ] Honeytokens deployed (canarytokens.org)
- [ ] HSTS preload submitted to hstspreload.org
- [ ] CAA record configured for domain

## ✅ Quick Commands

```bash
# Check for secrets
git ls-files | xargs grep -l "sk_live\|AKIA\|eyJh" 2>/dev/null

# Check .env in git
git ls-files | grep "\.env"

# Dependency audit
npm audit --audit-level=high

# Check for debug code
grep -r "console.log\|debugger\|TODO.*auth\|FIXME.*security" --include="*.{js,ts}" src/

# Check outdated deps
npm outdated
```
