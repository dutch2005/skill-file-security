# 🔥 Firebase Security Checklist

> Run before going live with a Firebase project.
> Generated by security-skill v1.0.0

---

## 🔴 Critical — Must Pass Before Launch

### Firestore Rules
- [ ] **No wildcard `allow all` rule** in production
  ```
  match /{document=**} { allow read, write: if true; }  // 🔴 NEVER
  ```
- [ ] All collections have explicit rules
- [ ] Authentication required for private data
- [ ] Ownership verified (users can only access their own documents)
- [ ] Admin operations protected by custom claim or separate collection
- [ ] Rules tested in Firebase Rules Simulator

### Storage Rules
- [ ] **No `allow read, write: if true`** on Storage rules
- [ ] File size limits enforced in rules
  ```
  allow write: if request.resource.size < 5 * 1024 * 1024;  // 5MB
  ```
- [ ] File type validation in rules
- [ ] Users can only access their own files

### Authentication
- [ ] Only necessary auth providers enabled
- [ ] Anonymous auth disabled (or limited) in production
- [ ] Email verification enforced for sensitive operations
- [ ] Custom claims not set by client-side code

---

## 🟠 High — Fix Before Launch

### API Key
- [ ] Firebase API key restricted in Google Cloud Console
  - APIs restricted to: Firebase-specific APIs only
  - HTTP referrers restricted to your domains
  - Not restricted to `*` (wildcard)
- [ ] App Check enabled and enforced in production

### Project Settings
- [ ] Project is not on default/legacy plan if using sensitive features
- [ ] Billing alerts configured
- [ ] Support email configured (used in security.txt)

### Functions (if used)
- [ ] Cloud Functions use least-privilege IAM roles
- [ ] Functions validate all inputs
- [ ] CORS configured (not wildcard `*` on authenticated endpoints)
- [ ] Rate limiting implemented in Functions

---

## 🟡 Medium — Fix Soon

### Monitoring
- [ ] Firebase Analytics configured
- [ ] Alerts set up for unusual usage (billing spikes)
- [ ] Firebase Performance Monitoring enabled

### Backups
- [ ] Firestore export scheduled (if data is critical)
- [ ] Backup location secured (not public Storage bucket)

### Keys & Secrets
- [ ] Service account key(s) tracked in `memory-security.md`
- [ ] Old service account keys rotated/deleted
- [ ] `firebase-adminsdk-*.json` in `.gitignore`

---

## 🔵 Low / Info

- [ ] Firebase Hosting headers configured (security-headers)
- [ ] Custom domain set up (not using firebaseapp.com)
- [ ] Preview channels secured (not shared publicly)

---

## ✅ Quick Verification Commands

```bash
# Test Firestore rules (requires firebase CLI)
firebase emulators:start

# Check what's deployed
firebase firestore:rules:get

# Check storage rules
firebase storage:rules:get
```

---

## 📋 Rules Testing Template

Test in Firebase Rules Simulator for each collection:
```
1. Read as unauthenticated user → should FAIL for private data
2. Read as authenticated user → should PASS for own data
3. Read other user's data → should FAIL
4. Write without required fields → should FAIL
5. Write with correct data → should PASS
6. Admin operations as regular user → should FAIL
```
