# 🐳 Docker Security Checklist

> Run before deploying Docker containers to production.
> Generated by security-skill v1.0.0

---

## 🔴 Critical — Must Pass Before Deploy

### Container Identity
- [ ] Container **does not run as root**
  ```bash
  docker inspect [container] | grep -i user
  # Should NOT be empty or "root"
  ```
- [ ] Non-root user created in Dockerfile with explicit UID/GID
- [ ] `USER` instruction present and not `USER root`

### Secrets
- [ ] **No secrets in Dockerfile** (ENV, ARG with values, RUN with exports)
- [ ] **No secrets in docker-compose.yml** (hardcoded environment values)
- [ ] `.dockerignore` excludes `.env` files and secrets
- [ ] Secrets injected at runtime via environment or Docker secrets

### Exposed Ports
- [ ] Only necessary ports exposed
- [ ] Database ports (5432, 3306, 27017, 6379) **not** published to host
- [ ] Debug ports (9229, 5000, etc.) not exposed in production

---

## 🟠 High — Fix Before Deploy

### Image Security
- [ ] Base image pinned to specific version (not `:latest`)
  ```dockerfile
  FROM node:20.11-alpine3.19  # ✅
  FROM node:latest            # 🔴
  ```
- [ ] Minimal base image used (alpine, slim, or distroless)
- [ ] Multi-stage build used (dev tools not in production image)
- [ ] Image scanned for vulnerabilities (`docker scout cves [image]`)

### Dockerfile Hardening
- [ ] `.dockerignore` file exists and covers sensitive files
- [ ] `npm ci` used instead of `npm install`
- [ ] Only production dependencies installed (`npm ci --only=production`)
- [ ] No `curl | bash` patterns during build

### docker-compose.yml
- [ ] `privileged: true` not used
- [ ] `network_mode: host` not used
- [ ] Resource limits defined (CPU + memory)
- [ ] Health checks configured
- [ ] `read_only: true` where possible

---

## 🟡 Medium — Fix Soon

### Runtime Security
- [ ] `--no-new-privileges` security option set
- [ ] Capabilities dropped (`cap_drop: ALL`)
- [ ] Only needed capabilities added back
- [ ] Filesystem mounted read-only where possible
- [ ] tmpfs for /tmp if writes needed

### Network
- [ ] Custom Docker networks used (not default bridge)
- [ ] Services communicate over internal Docker network (not host)
- [ ] External access only through reverse proxy (nginx/traefik)

### Logging
- [ ] Log driver configured (not logging sensitive data)
- [ ] Log rotation configured

---

## 🔵 Low / Info

- [ ] Image signed (Docker Content Trust)
- [ ] SBOM (Software Bill of Materials) generated
- [ ] Image tags follow semantic versioning

---

## ✅ Quick Checks

```bash
# Check if any container runs as root
docker ps -q | xargs -I{} docker inspect {} | grep '"User"'

# Check exposed ports
docker ps --format "table {{.Names}}\t{{.Ports}}"

# Scan image for vulnerabilities
docker scout cves myapp:latest

# Check for secrets in image layers
docker history myapp:latest --no-trunc

# Inspect security options
docker inspect myapp | jq '.[].HostConfig.SecurityOpt'
```
