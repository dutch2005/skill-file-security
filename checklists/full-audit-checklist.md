# 📋 Full Security Audit Checklist

> Comprehensive checklist covering all 25 security categories.
> Use when running /security-audit or doing a manual review.
> Generated by security-skill v1.0.0

---

## Category 01 — Secrets & Files (8%)

- [ ] `.gitignore` covers all sensitive patterns
- [ ] No `.env` files in git history
- [ ] No hardcoded secrets in source code
- [ ] `.env.example` exists (no real values)
- [ ] Pre-commit hook for secret scanning
- [ ] `gitleaks.toml` configured
- [ ] Secret rotation schedule defined

## Category 02 — Network & CORS (5%)

- [ ] CORS not using wildcard `*` in production
- [ ] CORS origin validated against allowlist
- [ ] Rate limiting on all API routes
- [ ] Strict rate limiting on auth endpoints (max 5/15min)
- [ ] HTTPS enforced with redirect from HTTP
- [ ] TLS 1.2+ only (no TLS 1.0/1.1)
- [ ] `NODE_TLS_REJECT_UNAUTHORIZED` not disabled
- [ ] Request body size limits set
- [ ] Request timeout configured

## Category 03 — HTTP Headers (5%)

- [ ] HSTS configured (max-age >= 31536000)
- [ ] Content-Security-Policy with nonces (no unsafe-inline)
- [ ] X-Frame-Options: DENY
- [ ] X-Content-Type-Options: nosniff
- [ ] Referrer-Policy: strict-origin-when-cross-origin
- [ ] Permissions-Policy (disabling unused APIs)
- [ ] Cross-Origin-Opener-Policy: same-origin
- [ ] Cross-Origin-Embedder-Policy: require-corp
- [ ] Server fingerprinting headers removed
- [ ] SRI on all CDN-loaded scripts

## Category 04 — Auth & Sessions (8%)

- [ ] Passwords hashed with bcrypt/argon2 (cost >= 12)
- [ ] Session regenerated after login (session fixation)
- [ ] Session cookies: httpOnly, secure, sameSite=strict
- [ ] Session inactivity timeout (30 min)
- [ ] Absolute session timeout (8 hours)
- [ ] Logout destroys server-side session
- [ ] CSRF protection (SameSite + Sec-Fetch or CSRF token)
- [ ] OAuth state parameter validated
- [ ] PKCE for public OAuth clients
- [ ] Password change requires current password
- [ ] Password reset tokens: 256-bit, 15-min expiry, single-use
- [ ] No security questions used
- [ ] MFA/2FA available on sensitive accounts
- [ ] Brute force protection (max 5 attempts/15 min)
- [ ] Account enumeration prevented (same error message always)

## Category 05 — Cryptography (6%)

- [ ] No MD5, SHA-1, DES, 3DES, RC4 used
- [ ] `crypto.randomBytes()` for all security-sensitive random values
- [ ] Random IV per encryption operation
- [ ] Timing-safe comparison for secrets (timingSafeEqual)
- [ ] PFS-compatible TLS ciphers (ECDHE)
- [ ] Buffer.alloc() used (not Buffer(), new Buffer())
- [ ] Sensitive data zeroed from memory after use

## Category 06 — JWT (5%)

- [ ] Explicit algorithm specified (`algorithms: ['RS256']`)
- [ ] 'none' algorithm rejected
- [ ] jku/jwk/kid headers validated against allowlist
- [ ] JWT secret >= 256 bits random
- [ ] Short expiry on access tokens (15 min)
- [ ] Sensitive data not in JWT payload
- [ ] Tokens stored in httpOnly cookies (not localStorage)
- [ ] Audience (aud) claim validated

## Category 07 — Database (7%)

- [ ] All SQL queries parameterized (no string concatenation)
- [ ] Firebase: no wildcard `allow all` rules
- [ ] Supabase: RLS enabled on all tables
- [ ] Supabase: service role key server-side only
- [ ] Least privilege DB user (not root/superuser)
- [ ] Database ports not publicly accessible
- [ ] SSL for database connections in production
- [ ] Redis: requirepass configured
- [ ] Redis: bound to localhost only
- [ ] Sensitive fields encrypted at rest (SSN, card, health)

## Category 08 — Deployment (5%)

- [ ] GitHub Actions: permissions set to minimum
- [ ] GitHub Actions: actions pinned to SHA
- [ ] No `pull_request_target` + checkout of fork code
- [ ] No secrets hardcoded in CI/CD config
- [ ] `NEXT_PUBLIC_*` prefix not used on secret values
- [ ] Dependabot configured
- [ ] Signed commits required on main branch
- [ ] API docs/swagger disabled in production

## Category 09 — Docker (3%)

- [ ] Container runs as non-root user
- [ ] Base image pinned (not `:latest`)
- [ ] `.dockerignore` excludes secrets
- [ ] Multi-stage build (no dev tools in prod)
- [ ] `privileged: false`
- [ ] Resource limits defined
- [ ] Database ports not published
- [ ] Image scanned for CVEs

## Category 10 — Protocols (3%)

- [ ] GraphQL introspection disabled in production
- [ ] GraphQL depth limit configured
- [ ] GraphQL complexity limit configured
- [ ] WebSocket: origin validated
- [ ] WebSocket: authentication on connection
- [ ] Webhook: HMAC signature verified
- [ ] SSE: authentication required

## Category 11 — Advanced Attacks (7%)

- [ ] SSRF: URL validation against allowlist before fetch
- [ ] SSTI: no user input in template rendering context
- [ ] Prototype Pollution: `__proto__` key rejected in merges
- [ ] Zip Slip: archive path validation before extraction
- [ ] Open Redirect: redirect URL validated against allowlist
- [ ] XXE: XML external entities disabled
- [ ] Mass Assignment: explicit field allowlist before DB write
- [ ] IDOR: ownership verified on all resource access

## Category 12 — Injections (6%)

- [ ] SQL: parameterized queries everywhere
- [ ] NoSQL: string coercion before MongoDB queries
- [ ] Command Injection: execFile (not exec) with arg array
- [ ] Log Injection: newlines stripped from logged user input
- [ ] Host Header Injection: APP_URL used (not req.headers.host)
- [ ] ReDoS: no nested quantifiers in user-facing regexes
- [ ] JSONP: callback validated as valid identifier

## Category 13 — Race Conditions (4%)

- [ ] Atomic DB operations for redemption/purchase flows
- [ ] DB transactions for multi-step financial operations
- [ ] Distributed lock for inventory/limited resources
- [ ] No integer overflow in price/quantity calculations
- [ ] Server-side price lookup (not client-provided)
- [ ] All async handlers wrapped in try/catch

## Category 14 — File Upload (3%)

- [ ] MIME type validated from file bytes (not extension)
- [ ] File size limits enforced
- [ ] Executable extensions blocked
- [ ] Files stored outside web root
- [ ] Filenames randomized (not original)
- [ ] Path traversal in filename prevented
- [ ] EXIF data stripped from images
- [ ] Symlink attacks prevented

## Category 15 — DNS & Email (3%)

- [ ] SPF record configured (`-all`)
- [ ] DKIM enabled (from email provider)
- [ ] DMARC configured (`p=quarantine` or `p=reject`)
- [ ] CAA record configured
- [ ] No dangling CNAME records (subdomain takeover risk)
- [ ] TLS certificate not expired
- [ ] `robots.txt` doesn't expose sensitive paths

## Category 16 — Supply Chain (5%)

- [ ] `npm audit` passes (no critical/high CVEs)
- [ ] Lockfile committed to git
- [ ] `npm ci` used in CI (not `npm install`)
- [ ] Dependabot configured
- [ ] No packages with malicious `postinstall` scripts
- [ ] Scoped packages for private dependencies
- [ ] License check passes for commercial use

## Category 17 — Mobile (2%)

- [ ] No secrets in mobile JS/Dart bundle
- [ ] Sensitive data in SecureStore (not AsyncStorage)
- [ ] Deeplink parameters validated server-side
- [ ] No PII/tokens in console.log
- [ ] `allowBackup: false` in Android config

## Category 18 — Compliance (4%)

- [ ] PII not in logs
- [ ] PII not in API responses (`password`, `creditCard`, etc.)
- [ ] PII not in URLs (query params)
- [ ] Right to erasure implemented (delete endpoint)
- [ ] Cookie consent before non-essential cookies (GDPR)
- [ ] Privacy policy page exists
- [ ] If healthcare: PHI access audit log
- [ ] If payments: raw card data never stored/logged

## Category 19 — Monitoring (3%)

- [ ] Security events logged (login, logout, auth failures)
- [ ] Errors logged server-side, generic message to client
- [ ] Honeytokens deployed
- [ ] `security.txt` at `/.well-known/security.txt`
- [ ] Directory listing disabled
- [ ] Secret rotation schedule tracked in memory-security.md

## Category 20 — Serverless (2%)

- [ ] Function timeout configured
- [ ] Lambda: IAM role least privilege
- [ ] No secrets in module-scope variables (shared across invocations)
- [ ] Event/request inputs validated

## Category 21 — Source Code (7%)

- [ ] No `eval()` or `new Function()` with user input
- [ ] No `innerHTML =` with user input (use textContent)
- [ ] No `exec(userInput)` (use execFile with array)
- [ ] No debug endpoints in production
- [ ] Source maps disabled in production
- [ ] No TODO/FIXME comments about security issues
- [ ] Null checks before property access on DB results
- [ ] No Trojan Source Unicode bidirectional characters

## Category 22 — AI/LLM (3%)

- [ ] AI API keys server-side only (no NEXT_PUBLIC_)
- [ ] Prompt injection guardrails in system prompt
- [ ] External content marked as untrusted in prompts
- [ ] LLM output validated against schema before use
- [ ] Tool/function permissions minimal
- [ ] RAG: user-scope filtering on vector DB queries

## Category 23 — Bot & DDoS (3%)

- [ ] Multi-layer rate limiting (global + per-endpoint)
- [ ] Redis-backed rate limiting (distributed)
- [ ] Request body size limits
- [ ] Request timeout configured
- [ ] Honeypot fields on forms

## Category 24 — Browser APIs (2%)

- [ ] WebRTC: TURN relay configured, no plain STUN
- [ ] Service Worker: auth requests never cached
- [ ] IndexedDB: sensitive data encrypted
- [ ] All data cleared from IndexedDB on logout
- [ ] postMessage: origin validated before handling

## Category 25 — Advanced Security L3 (2%)

- [ ] No Unicode bidirectional control characters in source
- [ ] Unicode normalized before comparison (NFKC)
- [ ] `target="_blank"` links have `rel="noopener noreferrer"`
- [ ] Symlink attacks prevented in file operations
- [ ] Session variable used for single purpose (no puzzling)
- [ ] Content-Type validated on incoming API requests

| Score | Status |
|---|---|
| 90-100 | 🟢 Excellent — maintain and monitor |
| 70-89 | 🟡 Good — fix highs and mediums |
| 50-69 | 🟠 Needs work — prioritize criticals |
| 0-49 | 🔴 Critical — stop and fix now |
